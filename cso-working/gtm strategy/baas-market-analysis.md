## 2. TAM / SAM / SOM 분석

### 2.1 TAM (Total Addressable Market)

**전체 BaaS 시장**

* **2025년:** $5B ~ $18B (정의에 따라 상이)
* **2030년:** $15B ~ $50B
* **CAGR:** 17% ~ 26%

보수적 기준으로 **2025년 $5B, 2030년 $15B** 적용

### 2.2 SAM (Serviceable Available Market)

**AI-Native / 바이브코딩 개발자를 위한 BaaS**

* AI Agent 시장: $7.38B (2025) → $103.6B (2032)
* 바이브코딩 도구 사용자 증가 추세 (Cursor, v0.dev, Bolt.new 등)
* 추정: TAM의 15-20%가 AI-Native 개발자 타겟

**2025년 SAM: $0.75B ~ $1.0B**

### 2.3 SOM (Serviceable Obtainable Market)

**초기 타겟 시장 (1-2년 내 획득 가능)**

* 아시아 태평양 지역 스타트업/중소기업
* 바이브코딩 얼리어답터
* AI-First 개발 방식 채택 기업

**2025년 SOM: $30M ~ $50M** (SAM의 3-5%)

## 3. 시장 성장 동인

### 3.1 주요 성장 요인

1. **클라우드 컴퓨팅 채택 증가**
   - 기업들의 클라우드 우선 전략 확대

2. **빠른 애플리케이션 개발 수요**
   - Time-to-market 단축 필요성
   - MVP 빠른 검증 트렌드

3. **바이브코딩 트렌드**
   - 비개발자/로우코드 개발자 증가
   - AI 어시스턴트 활용 개발 확산

4. **스타트업/SMB 성장**
   - 린 인프라 솔루션 수요
   - 초기 투자 비용 최소화 니즈

### 3.2 지역별 성장률

* **북미:** 가장 큰 시장 점유율
* **아시아 태평양:** 가장 빠른 성장률 (모바일/온라인 결제 증가)

## 4. 주요 플레이어 분석

### 4.1 시장 리더

| 플랫폼 | 특징 | 강점 |
|--------|------|------|
| **Firebase (Google)** | NoSQL, 실시간 동기화 | 250만+ 앱 지원, 쉬운 사용성 |
| **AWS Amplify** | AWS 생태계 연동 | 엔터프라이즈 스케일, 40% 성장 (2023) |
| **Supabase** | PostgreSQL 기반 오픈소스 | 80,000+ GitHub 스타, 개발자 친화적 |

### 4.2 신흥 플레이어

* **Appwrite:** 셀프호스팅/클라우드 옵션
* **Nhost:** PostgreSQL + GraphQL
* **Xano:** 노코드 백엔드 + AI 워크플로우
* **PocketBase:** 경량 싱글 바이너리

### 4.3 경쟁 구도 특징

**Firebase 이탈 트렌드:**
- 벤더 락인 우려
- 예측 불가능한 비용 구조
- Google Cloud 종속성

→ 오픈소스/유연한 대안 (Supabase, Appwrite) 부상

## 5. 시사점

### 5.1 시장 기회

1. **AI-Native BaaS 포지션 확보**
   - 기존 BaaS는 AI Agent 개발에 최적화되지 않음
   - 바이브코딩 친화적 기능 차별화 가능

2. **오픈소스 트렌드 활용**
   - Firebase 대안 수요 증가
   - 투명한 가격 구조 선호

3. **아시아 시장 선점**
   - 가장 빠른 성장률
   - 로컬 경쟁자 부재

### 5.2 도전 과제

1. 대형 클라우드 업체 (Google, AWS)와의 경쟁
2. 개발자 커뮤니티 구축 필요
3. 안정성 및 스케일 증명
