## 문제 해결 사이클

### 1. PDCA (Plan-Do-Check-Act) - 데밍 사이클

**개요**: W. Edwards Deming이 발전시킨 지속적 개선을 위한 4단계 반복 사이클

**단계별 프로세스**:

| 단계 | 활동 | 핵심 질문 |
|------|------|-----------|
| **Plan (계획)** | 목표 설정, 문제 정의, 가설 수립, 실행 계획 | "무엇을 달성하려 하는가?" |
| **Do (실행)** | 소규모 테스트, 데이터 수집, 계획 실행 | "계획대로 실행했는가?" |
| **Check (검토)** | 결과 분석, 예상과 비교, 교훈 도출 | "기대한 결과가 나왔는가?" |
| **Act (조치)** | 표준화 또는 개선, 다음 사이클 준비 | "어떻게 개선할 것인가?" |

**적용 사례**:
- 제조업 품질 관리
- 소프트웨어 개발 스프린트 회고
- 비즈니스 프로세스 개선

**장점**: 단순하고 이해하기 쉬움, 지속적 개선 문화 형성
**한계**: 복잡한 문제에는 너무 단순할 수 있음

---

### 2. OODA Loop (Observe-Orient-Decide-Act)

**개요**: 미 공군 전략가 John Boyd가 개발한 신속한 의사결정 프레임워크

**단계별 프로세스**:

```
┌─────────┐     ┌─────────┐     ┌─────────┐     ┌─────────┐
│ Observe │ ──> │ Orient  │ ──> │ Decide  │ ──> │   Act   │
│  관찰   │     │  판단   │     │  결정   │     │  행동   │
└─────────┘     └─────────┘     └─────────┘     └─────────┘
     ↑                                              │
     └──────────────────────────────────────────────┘
```

| 단계 | 설명 | 핵심 활동 |
|------|------|-----------|
| **Observe (관찰)** | 환경과 상황에서 정보 수집 | 데이터 수집, 시장 모니터링, 경쟁사 분석 |
| **Orient (판단)** | 정보를 맥락에 맞게 해석 | 패턴 인식, 편향 인식, 상황 분석 |
| **Decide (결정)** | 가능한 행동 방침 결정 | 옵션 평가, 자원 배분, 우선순위 설정 |
| **Act (행동)** | 결정 실행 및 환경 변화 | 계획 실행, 결과 모니터링 |

**적용 사례**:
- 스타트업 전략 수립
- 위기 관리
- 경쟁 시장에서의 신속한 대응
- 사이버 보안 대응

**장점**: 빠른 의사결정에 적합, 경쟁 환경에서 효과적
**한계**: 깊은 분석보다 속도 중시

### 3. DMAIC (Define-Measure-Analyze-Improve-Control)

**개요**: Six Sigma 방법론의 핵심 문제 해결 사이클

**단계별 프로세스**:

| 단계 | 목적 | 주요 도구 |
|------|------|-----------|
| **Define (정의)** | 문제와 목표를 명확히 정의 | 프로젝트 헌장, SIPOC, VOC |
| **Measure (측정)** | 현재 상태를 데이터로 측정 | 데이터 수집 계획, 측정 시스템 분석 |
| **Analyze (분석)** | 근본 원인 식별 | 5 Whys, 피쉬본 다이어그램, 회귀 분석 |
| **Improve (개선)** | 해결책 개발 및 구현 | 브레인스토밍, 파일럿 테스트, DOE |
| **Control (통제)** | 개선 상태 유지 | 관리도, SOP, 모니터링 계획 |

**적용 사례**:
- 제조 공정 개선
- 서비스 품질 향상
- 비용 절감 프로젝트
- 고객 만족도 개선

**장점**: 데이터 기반 의사결정, 체계적 접근
**한계**: 시간과 자원이 많이 소요됨

---

### 4. A3 Thinking (도요타)

**개요**: Toyota에서 개발한 A3 용지 한 장에 문제 해결 과정을 담는 방법론

**구성 요소**:

```
┌─────────────────────────────────────────────────────┐
│                    A3 보고서                         │
├──────────────────────┬──────────────────────────────┤
│ 1. 배경/맥락         │ 5. 대책                      │
│ 2. 현재 상태         │ 6. 실행 계획                 │
│ 3. 목표/목표 상태    │ 7. 후속 조치                 │
│ 4. 근본 원인 분석    │                              │
└──────────────────────┴──────────────────────────────┘
```

**단계별 프로세스**:

1. **배경 설명**: 왜 이 문제가 중요한가?
2. **현재 상태**: 지금 무엇이 일어나고 있는가?
3. **목표 설정**: 어디로 가고 싶은가?
4. **근본 원인 분석**: 왜 이 문제가 발생하는가?
5. **대책 수립**: 어떻게 해결할 것인가?
6. **실행 계획**: 누가, 언제, 무엇을 할 것인가?
7. **후속 조치**: 어떻게 성공을 확인할 것인가?

**적용 사례**:
- 린 제조
- 프로젝트 제안서
- 전략 계획
- 팀 문제 해결

**장점**: 간결함, 시각적 커뮤니케이션, 협업 촉진
**한계**: 복잡한 문제를 단순화할 위험

---

### 5. 8D Problem Solving (8 Disciplines)

**개요**: Ford에서 개발한 팀 기반 문제 해결 방법론

**8단계 프로세스**:

| D | 단계명 | 활동 |
|---|--------|------|
| D0 | **준비** | 문제 인식, 긴급 대응 조치 |
| D1 | **팀 구성** | 적합한 인력으로 팀 구성 |
| D2 | **문제 정의** | 5W2H로 문제 상세화 |
| D3 | **임시 조치** | 고객 보호를 위한 즉각 대응 |
| D4 | **근본 원인 분석** | 5 Whys, 피쉬본 등 활용 |
| D5 | **영구 대책 수립** | 해결책 개발 및 검증 |
| D6 | **영구 대책 실행** | 해결책 구현 및 검증 |
| D7 | **재발 방지** | 시스템/프로세스 개선 |
| D8 | **팀 공로 인정** | 성과 공유 및 축하 |

**적용 사례**:
- 제품 품질 문제
- 고객 불만 처리
- 공급망 문제
- 안전 사고 분석

**장점**: 체계적, 팀 기반 접근, 재발 방지 강조
**한계**: 시간이 많이 소요됨

---

### 6. IDEAL 모델

**개요**: 소프트웨어 프로세스 개선을 위해 SEI에서 개발한 모델

**단계별 프로세스**:

| 단계 | 설명 | 주요 활동 |
|------|------|-----------|
| **Initiating (시작)** | 개선 필요성 인식 | 스폰서십 확보, 인프라 구축 |
| **Diagnosing (진단)** | 현재 상태 평가 | 갭 분석, 우선순위 설정 |
| **Establishing (수립)** | 개선 전략 수립 | 목표 설정, 계획 수립 |
| **Acting (실행)** | 개선 활동 수행 | 파일럿, 배포, 교육 |
| **Learning (학습)** | 교훈 도출 | 성과 측정, 지식 공유 |

**적용 사례**:
- 조직 프로세스 개선
- 소프트웨어 개발 프로세스 성숙도 향상
- 변화 관리

**장점**: 조직 변화에 적합, 학습 강조
**한계**: 대규모 개선에 더 적합

### 7. 근본 원인 분석 (Root Cause Analysis)

#### 7.1 5 Whys

**개요**: "왜?"를 반복하여 근본 원인에 도달하는 기법

**예시**:
```
문제: 웹사이트가 다운되었다
  왜? → 서버가 과부하되었다
  왜? → 동시 접속자가 급증했다
  왜? → 마케팅 캠페인이 예상보다 성공했다
  왜? → 인프라 확장 계획이 없었다
  왜? → 마케팅-IT 간 커뮤니케이션이 부족했다
근본 원인: 부서 간 협업 프로세스 부재
```

**장점**: 단순하고 빠름, 누구나 사용 가능
**한계**: 복잡한 문제에는 부족할 수 있음

#### 7.2 피쉬본 다이어그램 (Ishikawa Diagram)

**개요**: 원인을 범주화하여 시각적으로 분석하는 도구

**6M 범주** (제조업):
- **Man (인력)**: 기술, 훈련, 경험
- **Machine (기계)**: 장비, 도구, 기술
- **Material (재료)**: 원자재, 부품
- **Method (방법)**: 절차, 프로세스
- **Measurement (측정)**: 검사, 데이터
- **Mother Nature (환경)**: 조건, 환경

**8P 범주** (서비스업):
- Price, Promotion, People, Process, Place, Product, Physical Evidence, Performance

## 컨설팅 프레임워크

### 1. MECE (Mutually Exclusive, Collectively Exhaustive)

**개요**: 문제를 겹치지 않고(ME) 빠짐없이(CE) 분해하는 원칙

**적용 방법**:

```
전체 매출
├── 지역별 (아시아 / 유럽 / 미주)      ← MECE
├── 제품별 (A / B / C)                  ← MECE
└── 채널별 (온라인 / 오프라인)          ← MECE

❌ Non-MECE 예시:
└── 고객별 (신규 / 기존 / VIP)          ← VIP가 신규/기존과 겹침
```

**활용 사례**:
- 문제 구조화
- 시장 세분화
- 원인 분석
- 옵션 평가

**장점**: 논리적 구조화, 누락 방지
**한계**: 복잡한 상호작용을 단순화할 수 있음

---

### 2. Issue Tree (이슈 트리)

**개요**: 핵심 질문을 MECE하게 하위 이슈로 분해하는 구조

**유형**:

#### 2.1 Why Tree (원인 분석)
```
수익이 감소한 이유는?
├── 매출이 감소했는가?
│   ├── 판매량이 줄었는가?
│   └── 가격이 하락했는가?
└── 비용이 증가했는가?
    ├── 고정비가 증가했는가?
    └── 변동비가 증가했는가?
```

#### 2.2 How Tree (해결책 도출)
```
수익을 어떻게 개선할 것인가?
├── 매출을 어떻게 증가시킬 것인가?
│   ├── 신규 고객을 어떻게 확보할 것인가?
│   └── 기존 고객 지출을 어떻게 늘릴 것인가?
└── 비용을 어떻게 절감할 것인가?
    ├── 운영 효율을 어떻게 높일 것인가?
    └── 불필요한 비용을 어떻게 제거할 것인가?
```

**적용 프로세스**:
1. 핵심 질문 정의
2. MECE 원칙으로 1차 분해
3. 각 가지를 추가 분해
4. 분석 가능한 수준까지 진행
5. 우선순위 설정

---

### 3. Hypothesis-Driven Approach (가설 주도 접근법)

**개요**: 가설을 먼저 세우고 데이터로 검증하는 방식

**프로세스**:

```
문제 정의 → 초기 가설 수립 → 이슈 트리 구성 → 분석 설계 → 데이터 수집 → 가설 검증 → 결론 도출
```

**가설의 특성**:
- **구체적**: 모호하지 않음
- **검증 가능**: 데이터로 증명 가능
- **실행 가능**: 행동으로 연결 가능

**예시**:
```
문제: 고객 이탈률이 높다

가설 1: "고객 서비스 응답 시간이 느려서 이탈률이 높다"
  → 검증: 응답 시간과 이탈률 상관관계 분석

가설 2: "경쟁사 대비 가격이 높아서 이탈률이 높다"
  → 검증: 가격 민감도 분석, 경쟁사 비교
```

**장점**: 효율적 분석, 80/20 원칙 적용
**한계**: 잘못된 초기 가설로 시간 낭비 가능

### 4. Pyramid Principle (피라미드 원칙)

**개요**: Barbara Minto가 McKinsey에서 개발한 구조화된 커뮤니케이션 프레임워크

**구조**:

```
           ┌─────────────┐
           │  핵심 메시지  │  ← 결론 먼저 (BLUF)
           │  (So What?) │
           └──────┬──────┘
                  │
    ┌─────────────┼─────────────┐
    │             │             │
┌───┴───┐   ┌─────┴─────┐   ┌───┴───┐
│ 근거 1 │   │   근거 2   │   │ 근거 3 │  ← MECE 지원 논거
└───┬───┘   └─────┬─────┘   └───┬───┘
    │             │             │
  세부사항      세부사항       세부사항
```

**핵심 원칙**:
1. **SCQ 프레임워크**: Situation(상황) - Complication(문제) - Question(질문)
2. **BLUF**: Bottom Line Up Front (결론 먼저)
3. **수직적 관계**: 상위 메시지는 하위 메시지의 "So What?"
4. **수평적 관계**: 동일 레벨은 논리적으로 연결 (귀납/연역)

**적용 예시**:
```
핵심 메시지: "신규 시장 진출을 권고합니다"

근거 1: 시장 기회가 크다
  - 시장 규모 $10B
  - 연평균 성장률 15%

근거 2: 경쟁이 제한적이다
  - 주요 플레이어 3개
  - 진입 장벽 낮음

근거 3: 우리 역량이 적합하다
  - 기존 기술 활용 가능
  - 파트너십 기회 존재
```

### 5. 80/20 원칙 (파레토 원칙)

**개요**: 결과의 80%는 원인의 20%에서 발생한다는 원칙

**적용 사례**:
- 매출의 80%는 고객의 20%에서 발생
- 문제의 80%는 원인의 20%에서 발생
- 성과의 80%는 활동의 20%에서 발생

**컨설팅 활용**:
- 분석 우선순위 설정
- 자원 배분 최적화
- "최소 노력으로 최대 영향" 영역 식별

## 전략 분석 프레임워크

### 1. Porter's Five Forces (포터의 5가지 힘)

**개요**: 산업의 경쟁 강도와 수익성을 분석하는 프레임워크

```
                    ┌─────────────┐
                    │  신규 진입자 │
                    │   의 위협    │
                    └──────┬──────┘
                           │
┌─────────────┐    ┌───────┴───────┐    ┌─────────────┐
│ 공급자의    │───►│   산업 내     │◄───│  구매자의   │
│ 교섭력     │    │   경쟁 강도    │    │  교섭력    │
└─────────────┘    └───────┬───────┘    └─────────────┘
                           │
                    ┌──────┴──────┐
                    │   대체재의   │
                    │    위협     │
                    └─────────────┘
```

**각 요소 분석**:

| 요소 | 주요 분석 포인트 |
|------|------------------|
| **신규 진입자** | 진입 장벽, 규모의 경제, 자본 요구, 브랜드 충성도 |
| **공급자 교섭력** | 공급자 집중도, 대체 공급원, 전환 비용 |
| **구매자 교섭력** | 구매자 집중도, 가격 민감도, 제품 차별화 |
| **대체재 위협** | 대체재 가용성, 상대적 가격/성능 |
| **산업 내 경쟁** | 경쟁자 수, 성장률, 고정비, 차별화 |

---

### 2. SWOT 분석

**개요**: 내부 강점/약점과 외부 기회/위협을 분석하는 프레임워크

```
           내부 요인              외부 요인
        ┌─────────────┐      ┌─────────────┐
긍정적  │  Strengths  │      │Opportunities│
        │    강점     │      │    기회     │
        └─────────────┘      └─────────────┘
        ┌─────────────┐      ┌─────────────┐
부정적  │  Weaknesses │      │   Threats   │
        │    약점     │      │    위협     │
        └─────────────┘      └─────────────┘
```

**TOWS 매트릭스** (전략 도출):

| | Strengths | Weaknesses |
|---|---|---|
| **Opportunities** | SO 전략: 강점으로 기회 활용 | WO 전략: 약점 극복해 기회 활용 |
| **Threats** | ST 전략: 강점으로 위협 대응 | WT 전략: 약점 최소화, 위협 회피 |

---

### 3. PESTEL 분석

**개요**: 외부 거시 환경을 6가지 요소로 분석

| 요소 | 분석 포인트 |
|------|-------------|
| **Political (정치)** | 정부 정책, 규제, 정치 안정성, 무역 정책 |
| **Economic (경제)** | GDP, 인플레이션, 환율, 금리, 실업률 |
| **Social (사회)** | 인구통계, 문화, 라이프스타일, 교육 수준 |
| **Technological (기술)** | R&D, 자동화, 디지털화, 혁신 속도 |
| **Environmental (환경)** | 기후 변화, 지속가능성, 환경 규제 |
| **Legal (법률)** | 노동법, 소비자 보호, 지적재산권, 안전 규제 |

---

### 4. Value Chain Analysis (가치 사슬 분석)

**개요**: 기업 활동을 분해하여 가치 창출과 비용 구조를 분석

```
┌─────────────────────────────────────────────────────┐
│              지원 활동 (Support Activities)          │
├─────────────────────────────────────────────────────┤
│ 기업 인프라 (경영, 재무, 법무, 품질관리)              │
├─────────────────────────────────────────────────────┤
│ 인적 자원 관리 (채용, 교육, 보상)                     │
├─────────────────────────────────────────────────────┤
│ 기술 개발 (R&D, 프로세스 개선, 제품 설계)             │
├─────────────────────────────────────────────────────┤
│ 조달 (구매, 공급업체 관리)                            │
└─────────────────────────────────────────────────────┘

┌─────────┬─────────┬─────────┬─────────┬─────────┐
│ 내부    │  운영   │  외부   │ 마케팅  │ 서비스  │ ─► 이윤
│ 물류    │         │  물류   │ & 판매  │         │
└─────────┴─────────┴─────────┴─────────┴─────────┘
              본원 활동 (Primary Activities)
```

---

### 5. 3C's 분석

**개요**: 전략적 의사결정을 위한 3가지 핵심 요소 분석

```
        ┌─────────┐
        │Customer │
        │  고객   │
        └────┬────┘
             │
    ┌────────┴────────┐
    │                 │
┌───┴───┐        ┌────┴────┐
│Company│        │Competitor│
│ 자사  │        │  경쟁사  │
└───────┘        └─────────┘
```

| 요소 | 분석 포인트 |
|------|-------------|
| **Customer** | 니즈, 세그먼트, 구매 행동, 의사결정 과정 |
| **Company** | 핵심 역량, 자원, 전략, 성과 |
| **Competitor** | 전략, 강점/약점, 시장 점유율, 예상 대응 |

---

### 6. 4P/7P Marketing Mix

**4P** (제품 마케팅):
- **Product (제품)**: 기능, 품질, 디자인, 브랜딩
- **Price (가격)**: 가격 전략, 할인, 지불 조건
- **Place (유통)**: 채널, 위치, 물류, 재고
- **Promotion (촉진)**: 광고, PR, 판촉, 영업

**7P** (서비스 마케팅 - 3P 추가):
- **People (사람)**: 직원, 고객 서비스
- **Process (프로세스)**: 서비스 전달 과정
- **Physical Evidence (물적 증거)**: 환경, 시설

---

### 7. Ansoff Matrix (성장 전략 매트릭스)

**개요**: 제품-시장 조합에 따른 4가지 성장 전략

```
              기존 제품         신규 제품
           ┌─────────────┬─────────────┐
기존 시장  │ 시장 침투    │ 제품 개발    │
           │ (저위험)     │ (중위험)     │
           ├─────────────┼─────────────┤
신규 시장  │ 시장 개발    │ 다각화       │
           │ (중위험)     │ (고위험)     │
           └─────────────┴─────────────┘
```

---

### 8. BCG Matrix

**개요**: 사업 포트폴리오를 시장 성장률과 상대적 시장 점유율로 분류

```
              높은 시장 점유율    낮은 시장 점유율
           ┌─────────────────┬─────────────────┐
높은       │     Stars       │   Question      │
성장률     │     (스타)       │   Marks (?)     │
           │   투자 & 성장    │   선택적 투자    │
           ├─────────────────┼─────────────────┤
낮은       │   Cash Cows     │     Dogs        │
성장률     │   (캐시카우)     │     (도그)      │
           │   수확 & 유지    │   정리/철수      │
           └─────────────────┴─────────────────┘
```

---

### 9. McKinsey 7S Framework

**개요**: 조직 효과성을 7가지 요소로 분석

**Hard S (하드 요소)** - 변경 용이:
- **Strategy (전략)**: 경쟁 우위 달성 계획
- **Structure (구조)**: 조직 구조
- **Systems (시스템)**: 일상 프로세스와 절차

**Soft S (소프트 요소)** - 변경 어려움:
- **Shared Values (공유 가치)**: 핵심 가치와 문화
- **Style (스타일)**: 리더십 스타일
- **Staff (인력)**: 직원과 역량
- **Skills (기술)**: 조직의 핵심 역량

### 프레임워크 조합 사용

복잡한 프로젝트에서는 여러 프레임워크를 조합하여 사용:

1. **전략 프로젝트**:
   - PESTEL → Five Forces → SWOT → Issue Tree → Pyramid Principle

2. **운영 개선 프로젝트**:
   - Value Chain → DMAIC → A3 → PDCA

3. **시장 진출 프로젝트**:
   - 3C's → SWOT → Ansoff → 4P/7P
